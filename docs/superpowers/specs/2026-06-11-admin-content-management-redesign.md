# 后台内容管理重构设计

## 背景

当前后台已有机构、剧集、分集、审核和 OSS 上传的基础能力，但页面交互割裂，列表信息不足，详情编辑依赖跳转，媒体展示直接暴露 path 或缺少真实可访问 URL。此次重构目标是在保留现有业务模型和前台视觉体验的前提下，把后台内容管理整理成统一、可维护、可扩展的运营工作台。

## 范围

本次实现完整闭环：

- 管理员可在后台新增机构，并选择创建后直接启用或待审核。
- 登录页提供机构注册入口，公开注册页仍用于机构自助申请。
- 剧集列表展示竖版海报、审核状态、上架状态，并提供独立审核、上架、下架按钮。
- 剧集详情通过右侧抽屉编辑，包含横版封面、竖版海报、剧名、短标语、剧情简介、分类和排序；审核与上架状态只展示，状态切换由列表行按钮独立完成。
- 分集列表展示视频预览容器，点击后用右侧抽屉编辑分集信息。
- 新建/编辑分集支持视频上传：前端直传 OSS，成功后将 object path 提交给后端。
- 后台 UI 统一为“深色侧栏 + 浅色内容区 + 列表 + 右侧抽屉”的交互模式。
- 重构上传组件，支持图片、视频、文件预览和替换。

不纳入本次：

- 不新增完整分类管理页面，分类使用固定枚举。
- 不在剧集详情维护或展示预告片。
- 不在后台维护分集观看权限，分集统一按会员内容处理。
- 不重构前台页面视觉。

## UI 与交互原则

后台采用统一的工作台模式：

- 侧栏保持深色品牌风格，用于主导航。
- 主内容区改为浅色专业后台界面，提高长时间管理可读性。
- 剧集、分集、机构页面都使用“列表 + 右侧抽屉”。
- 点击列表行打开详情抽屉；点击新增按钮打开新增抽屉。
- 抽屉底部固定“保存 / 取消”，内容编辑必须手动保存。
- 状态动作不跟随详情保存，列表行内独立触发。
- 移动端抽屉降级为全屏面板。

共享组件：

- `AdminDrawer`：标题、副标题、内容区、底部固定操作区、关闭确认、移动端全屏。
- `AdminListToolbar`：搜索、筛选分组、主操作按钮。
- `AdminStatusBadge`：统一状态文案和视觉。
- `AdminActionButton`：统一列表行状态操作按钮。
- `AdminMediaUpload`：统一上传入口，支持图片、视频、文件三类预览。
- `AdminMediaPreview`：统一列表和详情媒体展示，空值显示占位，不用裸 path 代替媒体。
- 固定分类多选组件。

## 状态设计

### 剧集审核状态

沿用 `reviewStatus`：

- `draft`：草稿
- `submitted`：待审核
- `approved`：已通过
- `rejected`：已驳回

审核动作显示规则：

- 只有 `reviewStatus=submitted` 显示“通过 / 驳回”。
- `draft` 和 `rejected` 显示“提交审核”。
- `approved` 不显示审核按钮。
- 驳回必须填写原因。

### 剧集上架状态

沿用 `releaseStatus`：

- `upcoming`：待上架
- `released`：已上架

上架动作显示规则：

- `reviewStatus=approved && releaseStatus=upcoming` 显示“上架”。
- `reviewStatus=approved && releaseStatus=released` 显示“下架”。
- 未审核通过的剧集不允许上架。
- 上架/下架只更新 `releaseStatus`，不和详情表单保存绑定。

### 分集发布状态

沿用 `Episode.status`：

- `draft`：草稿
- `published`：已发布

分集列表行提供独立状态按钮：

- `draft` 显示“发布”。
- `published` 显示“下架”或“设为草稿”。
- 按钮只更新分集发布状态和 `publishedAt`。

## 媒体 URL 设计

数据库仍只保存 OSS path，不保存签名 URL 或公开 URL。后台所有媒体展示都由服务端按当前管理员权限生成短时签名 URL。

后台接口返回时补充展示 URL：

- 剧集列表：`posterUrl` 优先展示竖版海报，没有时降级 `coverUrl`。
- 剧集详情：返回 `coverUrl`、`posterUrl`，不展示预告片。
- 分集列表：返回 `videoPreviewUrl`，用于视频预览容器。
- 分集详情：返回可播放视频 URL。
- 机构详情：营业执照 path 转为短时签名 URL 展示。

表单提交仍只传 path：

- `coverPath`
- `posterPath`
- `videoPath`
- `episode.coverPath`
- `businessLicensePath`

## 剧集管理

### 列表

剧集列表顶部包含两个筛选组：

- 上架状态：全部、待上架、已上架。
- 审核状态：全部、草稿、待审核、已通过、已驳回。

列表支持按剧名搜索。

列表行展示：

- 竖版海报。
- 剧名。
- 分类。
- 分集数。
- 审核状态。
- 上架状态。
- 更新时间。
- 分集入口。
- 审核和上架快捷按钮。

列表不展示短标语。

### 新建与详情抽屉

新建剧集和编辑剧集都使用 `AdminDrawer`。

字段：

- 剧名：数据库字段 `title`。
- 短标语：数据库字段 `subtitle`，选填。
- 剧情简介：数据库字段 `synopsis`。
- 横版封面：数据库字段 `coverPath`。
- 竖版海报：数据库字段 `posterPath`。
- 分类/题材：固定枚举多选。
- 审核状态和上架状态：只展示，不随抽屉保存提交；审核、上架、下架由列表行快捷按钮独立触发。
- 排序：数据库字段 `sortOrder`。

`slug` 不在后台表单展示。创建剧集时由服务端自动生成唯一 slug，数据库字段继续保留。

固定分类枚举：

- 都市
- 爱情
- 悬疑
- 古装
- 喜剧
- 仙侠
- 战争
- 惊悚

## 分集管理

从剧集列表行点击“分集”进入对应剧集的分集列表页。页面继续使用“列表 + 右侧抽屉”模式。

列表顶部展示：

- 当前剧名。
- 返回剧集列表。
- 搜索或状态筛选。
- 新建分集按钮。

列表行展示：

- 视频预览容器。
- 集数。
- 标题。
- 发布状态。
- 时长。
- 更新时间。
- 发布/下架快捷按钮。

视频预览容器使用后台签名 URL。列表不主动缓冲完整视频，只需要让运营能看到该集内容的可识别画面；点击分集打开抽屉后再加载可播放视频。

新建/编辑字段：

- 集数。
- 标题。
- 简介/摘要。
- 视频上传。
- 可选封面上传。
- 时长。
- 发布状态。

不展示观看权限字段。后端创建和更新时默认 `accessLevel=member`。

## 机构管理与注册入口

### 注册入口

`/admin/login` 增加明显的“机构注册”入口，跳转到现有 `/admin/register`。

`/admin/register` 保持公开注册语义：

- 机构自己提交资料。
- 默认创建待审核机构。
- 机构账号默认为 `pending`。

### 后台新增机构

后台 `/admin/organizations` 重构为列表 + 抽屉。

“新增机构”使用独立组件，不复用公开注册页组件。

字段：

- 机构名称。
- 联系人。
- 联系电话。
- 邮箱。
- 统一社会信用代码。
- 地址。
- 简介。
- 营业执照上传。
- 初始登录密码。
- 创建状态：直接启用 / 待审核。

创建状态规则：

- 直接启用：`organization.status=approved`，机构账号 `active`。
- 待审核：`organization.status=pending`，机构账号 `pending`。

机构列表行展示：

- 机构名称。
- 联系人。
- 联系电话。
- 状态。
- 创建时间。
- 审核动作。

只有 `status=pending` 的机构显示“通过 / 驳回”。已通过或已驳回不显示审核按钮。

点击机构行打开详情抽屉，支持查看与编辑基础资料。营业执照通过签名 URL 展示。

## 上传组件

`AdminMediaUpload` 替换现有基础 file input。

能力：

- 根据类型展示图片、视频或文件预览。
- 上传前展示明确的选择区域。
- 上传中展示加载状态。
- 上传成功后显示预览和 path。
- 支持替换。
- 支持清除可选字段。
- 不在日志或 UI 中展示签名 URL 全量内容。
- 仍使用 `/api/admin/uploads/policy` 获取直传策略。
- 直传 OSS 成功后把 object path 写入表单。

使用场景：

- 横版封面。
- 竖版海报。
- 分集视频。
- 分集可选封面。
- 营业执照。

## API 与服务端设计

沿用 API 返回规范：

成功：

```json
{
  "data": {}
}
```

失败：

```json
{
  "error": {
    "code": "AUTH_REQUIRED",
    "message": "请先登录"
  }
}
```

需要补充或调整：

- Admin 剧集列表和详情接口返回签名媒体 URL。
- Admin 分集列表和详情接口返回签名视频 URL。
- Admin 机构详情接口返回营业执照签名 URL。
- 剧集创建时服务端自动生成唯一 slug。
- 剧集上架/下架状态接口，只更新 `releaseStatus`。
- 分集发布/下架状态接口，只更新 `status` 和 `publishedAt`。
- 管理员创建机构接口支持 `initialStatus`。
- 剧集分类保存逻辑支持固定枚举多选，写入 `DramaGenre`。

权限和安全：

- 媒体签名 URL 只在已登录后台接口返回。
- 机构账号只能访问自己机构下的剧集和分集。
- 管理员可访问全部机构和剧集。
- 未审核通过的剧集不能上架。
- 审核中机构或剧集按现有规则限制编辑。
- 服务端继续校验上传 path 是否属于当前管理员允许范围。

## 错误处理

- 上传失败：组件内显示错误，保留原 path。
- 保存失败：抽屉内显示错误，不关闭抽屉。
- 状态切换失败：列表行动作区显示错误或页面级 toast。
- 驳回缺少原因：服务端返回校验错误，前端提示必须填写。
- 未通过审核尝试上架：服务端拒绝，前端按钮禁用并显示提示。
- 媒体签名失败：展示占位，不暴露内部错误或签名细节。

- 剧集 `releaseStatus` 可在审核通过后切换为待上架或已上架。

页面和静态测试：

- 登录页存在机构注册入口。
- 剧集页使用统一抽屉组件。
- 分集页使用统一抽屉组件。
- 机构页使用统一抽屉组件。
- 剧集列表展示竖版海报、审核状态、上架状态。
- 分集列表展示视频预览容器。
- 上传组件用于封面、海报、分集视频和营业执照。
- 后台页面不直接把 path 当成媒体 URL 使用。


## 迁移与兼容

- 不删除 `slug` 字段，只改为服务端自动生成。
- 不删除 `subtitle` 字段，后台 label 改为“短标语”。
- 不删除 `trailerPath` 字段，本次后台不维护预告片。
- 不删除 `accessLevel` 字段，本次后台隐藏并默认会员。
- 不新增“待上架”字段，使用现有 `releaseStatus=upcoming`。
- 保持现有前台 API 和页面视觉稳定。
